cli

概览​

dora-rs cli 客户端

用法: dora <COMMAND>

命令:
  check        检查协调器和守护程序是否正在运行
  graph        使用 mermaid.js 生成给定图形的可视化 使用 --open 打开浏览器
  build        运行在给定数据流提供的 build 命令
  new          生成一个新项目，节点(node) 或 算子(operator) 在 Rust, Python, C 或 C++ 之间选择语言
  up           生成一个协调器和一个守护程序
  destroy      销毁运行中的协调器和守护程序 如果某些数据流仍在运行，则它们将首先停止
  start        启动给定数据流路径 使用 --name 附加一个名称至运行中的数据流
  stop         停止给定数据流UUID 如果没有提供ID，您将在运行中的数据流之间选择
  list         列表出运行中的数据流
  logs         显示一个给定数据流和节点日志
  daemon       运行守护进程
  runtime      运行运行时
  coordinator  运行协调器
  help         打印本帮助信息或给定子命令的帮助信息

选项:
  -h, --help     打印帮助信息
  -V, --version  打印版本信息

up​

生成一个协调器和一个守护进程

用法: dora up [选项]

选项:
      --config <CONFIG>  
  -h, --help             打印帮助信息

new​

生成一个新项目，节点或算子。 在 Rust, Python, C 或 C++ 之间选择语言

用法: dora new [选项] <NAME> [路径]

参数:
  <NAME>  
  [路径]  

选项:
      --kind <KIND>  [默认: dataflow] [可能的选项: dataflow, operator, custom-node]
      --lang <LANG>  [默认: rust] [可能的选项: rust, python, c, cxx]
  -h, --help         打印帮助信息

start​

从给定数据流路径启动。 使用 --name 附加一个名称至运行中的数据流

用法: dora start [选项] <DATAFLOW>

参数:
  <DATAFLOW>  

选项:
      --name <NAME>  
      --attach       
      --hot-reload   
  -h, --help         打印帮助信息

list​

列出运行中的数据流

用法: dora list

选项:
  -h, --help  打印帮助信息

logs​

显示一个给定数据流和节点的日志

用法: dora logs [数据流] <NODE>

参数:
  [数据流]  
  <NODE>      

Options:
  -h, --help  打印帮助信息

check​

对运行中的协调器和守护进程检查

用法: dora check [选项]

选项:
      --dataflow <DATAFLOW>  
  -h, --help                 显示帮助信息

stop​

停止给定UUID的数据流。 如果没有提供 id，您将能够在运行中的数据流中选择。

用法: dora stop [选项] [UUID]

参数:
  [UUID]  

选项:
      --name <NAME>  
  -h, --help         显示帮助信息

destroy​

销毁运行中的协调器和守护进程。 如果一些数据流仍在运行，他们将被先停止。

用法: dora destroy [选项]

选项:
      --config <CONFIG>  
  -h, --help             显示帮助信息

graph​

对给定图使用 mermaid.js 生成可视化。 使用 --open 打开浏览器

用法: dora graph [选项] <DATAFLOW>

参数:
  <DATAFLOW>  

选项:
      --mermaid  
      --open     
  -h, --help     显示帮助信息

--version​

返回 dora 的当前版本

这个命令将显示 dora 的当前版本。
