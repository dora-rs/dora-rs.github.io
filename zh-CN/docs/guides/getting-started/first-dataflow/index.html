first-dataflow

sidebar_position: 3

开始

  1. 使用我们的 安装页面 安装 dora 二进制文件

  2. 创建一个新的数据流

    dora new abc_project --lang python
    cd abc_project

    这就创建了以下 abc_project 目录

    .
    ├── dataflow.yml
    ├── node_1
    │   └── node_1.py
    ├── op_1
    │   └── op_1.py
    └── op_2
        └── op_2.py

  3. 启动 dora-coordinator 和 dora-deamon

    dora up

  4. 启动您的数据流

    dora start dataflow.yml --name first-dataflow
    # Output: c95d118b-cded-4531-a0e4-cd85b7c3916c

    输出是一个随机生成的数据流实例的唯一ID，可以通过 dora CLI来控制它。 您可以使用 --name 选项为您的数据流设置一个特定名称。

  5. 您可以通过以下方式查看日志：

    dora logs custom-node_1

    在此示例中，输出将是：

    Node received:
        id: tick,
        value: 0 nulls,
        metadata: {'open_telemetry_context': ''}

  6. 停止您的数据流

    dora stop --name first-dataflow

    （在此处传递 dora start 返回的 ID 或名称。）

  7. 然后您可以添加或修改操作符或节点。 为了轻松添加节点，您可以再次使用 dora CLI ：

    • 运行 dora new --kind operator --lang rust <name> 建立一个新的Rust 操作符名称 <name>。
    • 运行 dora new --kind custom-node --lang rust <name> 建立一个新的自定义节点名称 <name>.

    您需要添加已创建的 操作符/节点 至您的数据流 YAML 文件。
